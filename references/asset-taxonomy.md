# Asset Taxonomy

Use this as a checklist; adapt to the repo.

## Canonical Masters

- `approved/logo.svg` or `approved/<brand>-logo.svg`
- `approved/logo.png` when generated raster is the approved visual source
- `approved/app-icon.png`
- `approved/material-board.png`
- `approved/color-palette.png`

## Product Runtime Assets

- app logo SVG used by components
- app icon PNG source
- macOS `.icns`
- Windows `.ico`
- Linux PNG icons at required sizes
- Electron/Tauri/native build icons
- mobile app icon
- adaptive foreground/background icon
- favicon
- splash icon/image
- onboarding/empty-state brand marks

## Brand/Marketing Assets

- README hero background
- social card / OpenGraph image
- release note image
- website hero image
- store listing icon/screenshots
- press kit logo variants

## Documentation Assets

- visual identity document
- brand kit inventory
- generation DAG
- prompt log / rationale
- accepted/rejected decision notes only when useful; do not keep rejected images in product paths

## Verification Targets

- dimensions and alpha channel
- small-size legibility
- light/dark background contrast
- text-free generated imagery unless post-composed text is intentional
- source path references in docs and build scripts
- stale old-brand filenames
